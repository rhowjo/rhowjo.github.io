The aim of this post will be to evaluate the practices of Corazón in comparison to the British Computer Society Code of Conduct (BCS Code of conduct, 2022) regarding the (harmless) security vulnerability discovered by a researcher in their medical device (Case: Medical implant risk analysis - ACM ethics 2021).

There are 4 parts to the code of conduct, and each will be considered in turn, with reference to specific codes.

Public Interest:

By the very nature of the implant device, Corazón complies with this part of the code as it demonstrates due regard for public health and wellbeing (1.a). Additionally, by having an accessible device that has an easy-to-use app, Corazon also promotes equal access to the benefits of IT (1.d).

Professional Competence and Integrity:

Instead of using proprietary techniques, using standard cryptographic algorithms demonstrates compliance with 2.a, only undertaking work within professional competence.

Duty to Relevant Authority:

By actively pursuing regulatory compliance to regulatory standards, Corazón demonstrates its due diligence to fulfilling obligations to the relevant regulations.

Duty to the Profession:

With on-going commitments to consult with researchers and openness of product performance and security by welcoming scrutiny, Corazón proves that it complies with all aspects of this code.

On this basis, Corazon shows exemplary practices regarding ethics. However, in lieu of examining this case in isolation, there is a call for greater regulatory frameworks as not all risks are accounted for in current standards for smart implant devices (Harmon et al., 2015) thus it remains to be seen whether this case would uphold the ethics if new regulations were put in place.

Task – Reasoning Quiz

This quiz sought to test knowledge of differentiating between deductive reasoning and inductive reasoning. Deductive reasoning is making an inference based on accepted facts or premise and for it to work, the premise must be true (Inductive vs. deductive vs. Abductive Reasoning). E.g. All humans have hearts. Rhiannon is a human. Thus, she must have a heart. On the other hand, for inductive reasoning, the premise is likely but not necessarily true based on observations (Hawthorne, 2018).

Through assessing multiple applications in the previous section, the literature review will then assess the many ways in which bias can factor into AI models.

    Task – Jaccard Coefficient

    For this task, we were asked to look at Jaccard Coefficient Calculations also known as Jaccard similarity. This is used to compute the similarity between objects.

    The formula is thus: JC=Ni/(Na+Nb−Ni)

    • Ni is the intersecting elements (how many elements are shared between the sets

    • Na is the number of elements in set A.

    • Nb is the number of elements in set B.

    In this scenario, elements will be instances of Y and P in the various columns.

    Jack and Mary:

    • Jack – Fever, Test-1 = 2 elements

    • Mary – Fever, Test-1, Test-3 = 3 elements

    • Intersecting Elements = 2

    • JC = 2/ (2+3−2) = 0.66

    Jack and Jim:

    • Jack – Fever, Test-1 = 2 elements

    • Jim – Fever, Cough = 2 elements

    • Intersecting Elements = 1

    • JC = 1/ (2+2-1) = 0.33

    Jim and Mary

    • Jim – Fever, Cough = 2 elements

    • Mary – Fever, Test-1, Test-3 = 3 elements

    • Intersecting Elements = 1

    • JC = 1/ (2+3-1) = 0.25

    Task - Gradeient Cost Function

    In this task, we were asked to observe changes in cost in the Jupyter Notebook provided in the seminar materials.

    Below I have collated information on how the gradient cost function works and the impact of the learning rate and iterations:

    A cost function is the measure of error in a prediction by an algorithm by indicating the difference between predicted values and actual values for a dataset (Banoula, 2023). A technique used to minimise the cost function is this gradient descent method (Team, 2022). The smaller the cost function, the more accurate the results of the predictive model.

    The learning rate in gradient descent, and in this demonstration of gradient cost function, controls the rate at which the model adapts to the problem (Brownlee, 2020). A lower learning rate will require more iterations as there are smaller changes after each iteration. Larger learning rates will have the opposite effect and require fewer iterations.

    If the learning rate is too large, however, the model can converge too rapidly and reach a suboptimal solution. If it’s too small, the process may take a very long time (Brownlee, 2020).

    Thus, it is important that the learning rate must be carefully selected for optimal outcomes.

    • Task – Model Performance Measurement

    In this task, I observed the changes of parameters affecting the AUC and R2 error. The Jupyter Notebook was provided in the seminar materials.

    The AUC (area under the curve or ROC curve) measures the are underneath an ROC curve. The ROC is a probability curve, and the AUC measures the degree of separability (Bhandari, 2023). Essentially, it measures how capable the model is at distinguish between different classes (Narkhede, 2021). The higher the AUC, the better the predictive model is.

    In the notebook, changing the number of features has varying impact on the AUC, and thus the predictive capabilities of the model in terms of accuracy.

    The R2 score is the proportion of variance in the dependent variable that is predictable from the independent variable. It can also be defined as the total variance explained by the model divided by the total variance (Kumar, 2023). High values indicate that there is perfect correlation between variables and low values indicate low correlation.

    A high R2 score generally indicates that the model is a good fit, although other considerations such as overfitting and underfitting should be taken into account.
